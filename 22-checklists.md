# Security Checklists

These checklists provide structured reminders
for common cybersecurity activities.
They are meant to support consistency and completeness,
not to replace understanding.

All items apply to learning and legal environments only.

---

## 1. General Security Checklist

- identify assets
- define scope and objectives
- understand threat model
- apply least privilege
- enable logging
- monitor continuously
- review regularly

---

## 2. Networking Checklist

- verify IP configuration
- confirm required ports only are open
- test basic connectivity (ping)
- verify routing paths
- confirm DNS resolution
- review firewall rules
- reduce exposed services

---

## 3. Web Application Checklist

- identify all endpoints
- review authentication logic
- review authorization logic
- validate server-side checks
- handle user input safely
- review error handling
- minimize exposed data

---

## 4. API Security Checklist

- authenticate all requests
- authorize every action
- validate object ownership
- limit data exposure
- implement rate limiting
- handle errors safely
- log API activity

---

## 5. Linux System Checklist

- keep system updated
- verify file permissions
- review user privileges
- limit sudo access
- monitor running processes
- review logs
- remove unused software

---

## 6. Windows System Checklist

- review user and admin accounts
- enforce UAC
- review NTFS permissions
- monitor services
- review event logs
- keep systems patched
- disable unnecessary services

---

## 7. Active Directory Checklist

- review domain admin accounts
- verify group memberships
- limit privileged access
- review GPOs
- monitor authentication logs
- protect Domain Controllers
- document trust relationships

---

## 8. Cloud Security Checklist

- review IAM permissions
- enforce least privilege
- secure storage access
- enable logging and monitoring
- review network rules
- protect management interfaces
- audit configurations regularly

---

## 9. Detection and Monitoring Checklist

- confirm logs are collected
- validate log sources
- create meaningful alerts
- reduce alert noise
- test detection rules
- review detection gaps
- document improvements

---

## 10. Incident Response Checklist

- confirm incident is real
- assess impact and scope
- contain affected systems
- preserve evidence
- eradicate root cause
- recover operations
- document lessons learned

---

## Key Takeaways

- checklists improve consistency
- they reduce missed steps
- they support structured thinking
- they do not replace expertise
- regular review keeps them relevant
