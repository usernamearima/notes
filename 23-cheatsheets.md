# Security Cheatsheets

These cheatsheets provide quick reference summaries
for common cybersecurity concepts and commands.
They are designed for fast recall, not deep explanation.

All content is for learning and legal environments only.

---

## 1. Networking Cheatsheet

### Common Ports
- 80 – HTTP
- 443 – HTTPS
- 22 – SSH
- 21 – FTP

### Basic Commands
```bash
ip a
ping 8.8.8.8
traceroute google.com
```
---

## 2. HTTP Cheatsheet
Methods:
GET – retrieve data
POST – send data
PUT – update data
DELETE – remove data

Common Status Codes:
- 200 – OK
- 301 / 302 – Redirect
- 401 – Unauthorized
- 403 – Forbidden
- 404 – Not Found
- 500 – Server Error

## 3. Linux Cheatsheet
Navigation:
pwd
ls
cd

Permissions:
r – read
w – write
x – execute

Permission Groups:
u – owner
g – group
o – others

## 5. Web Security Cheatsheet

authentication ≠ authorization
backend enforces security
frontend is not trusted
validate user input
log security events

---

## 6. API Security Cheatsheet

- authenticate every request
- authorize every action
- validate object ownership
- limit data exposure
- apply rate limiting

---

## 7. Active Directory Cheatsheet

- domain is the security boundary
- Domain Controllers are critical
- groups manage permissions
- GPOs enforce policy
-logs enable detection

---

## 8. Cloud Security Cheatsheet

- shared responsibility model
- IAM is critical
- misconfiguration is common risk
- logging is essential
- least privilege reduces impact

---

## 9. Incident Response Cheatsheet
IR Phases:
- Preparation
- Identification
- Containment
- Eradication
- Recovery
- Lessons Learned

---

## 10. Reporting Cheatsheet

- be clear and concise
- explain impact
- include evidence
- give actionable recommendations
- maintain professional tone

---

Key Takeaways:
- cheatsheets support fast recall
- they complement full notes
- consistency improves usability
- keep them short and practical
